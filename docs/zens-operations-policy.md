# Zens 自动运营策略

## 社区定位

Zens 是一个友善、真实、鼓励长期表达的开放社区。技术可以讨论，生活也可以分享；既欢迎完整教程，也欢迎真实经历、独立作品、问题求助和有价值的观点。

## 自动化原则

- 自动系统只使用“Zens运营”服务身份，不复用个人账号。
- 只读研究走 Agent 服务和 MySQL 从库；所有写入必须经过主站业务服务、权限、审核和审计链路。
- 默认只生成草稿。首批 30 篇正式内容必须由管理员逐篇批准。
- 同一任务必须带幂等键；重试不能产生重复草稿、帖子、评论或指标快照。
- 熔断开启时仍可分析、生成计划和保存草稿，但禁止发布帖子和写入评论。
- 每日最多发布 1 篇正式内容、自动回复 10 条；超过限额必须等待下一自然日或由管理员修改配置。

## 允许自动执行

- 分析站内近 7 天和 30 天的主题、搜索缺口、无人回复问题和持续讨论。
- 生成原创 Markdown 草稿、标题、摘要、标签、内部链接建议和配图建议。
- 生成“Zens 本周精选”和运营指标报告草稿。
- 对官方内容下的低风险事实型问题生成回复草稿。
- 记录任务状态、耗时、模型信息、输入来源、审批结果和失败原因。

## 必须人工确认

- 政治、医疗、法律、金融、未成年人、自伤、投诉纠纷等高风险主题。
- 商业广告、价格、优惠、抽奖、付费、合同、收益或效果承诺。
- 外部文章转载、版权不明确的图片或大段引用。
- 删除普通用户内容、封禁用户、私信营销和批量通知。
- 首批 30 篇正式发布，以及任何不满足置信度或原创性阈值的内容。

## 内容与版权

- 不复制其他社区的文章，不通过轻微改写规避版权。
- 外部资料只能用于事实核对和研究，发布内容必须有实质性原创表达。
- 引用必须给出来源、作者和链接；图片仅使用原创、自有 R2、明确授权或允许复用的素材。
- 不伪造亲身经历、用户反馈、数据、测试结果或社区共识。

## 审批状态

`CREATED → PENDING_APPROVAL → APPROVED → PUBLISHED`

草稿的人工拒绝分支为 `REJECTED`；发布任务失败记录在独立审计任务的 `FAILED` 状态中，草稿由事务回滚恢复为可安全复核的状态。所有状态变化必须记录操作者、时间、原因和幂等键。已经发布的内容不能通过重试再次发布。

## 灰度与停机

1. 第一阶段只运行分析、计划、草稿和指标，不发布、不评论。
2. 首批 30 篇经人工审批后，评估原创性、准确性、互动和投诉情况。
3. 仅对白名单板块逐步开放自动发布，仍保留每日限额与熔断。
4. 出现重复发布、事实错误、异常评论、投诉或鉴权异常时立即打开熔断。
