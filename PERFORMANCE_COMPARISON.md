## 🎓 最佳实践建议

### 1. 缓存使用原则
```java
// ✅ 好：高频读、低频写的数据
@Cacheable(value = "user:info", key = "#userId")
public User getById(String userId) { }

// ❌ 差：实时性要求高的数据
@Cacheable("order:realtime") // 不适合缓存
public Order getCurrentOrder() { }
```

### 2. 限流配置原则
```java
// ✅ 好：根据业务特点设置合理阈值
@RateLimit(limit = 10, windowSeconds = 60)  // 发帖
@RateLimit(limit = 100, windowSeconds = 60) // 查询

// ❌ 差：一刀切
@RateLimit(limit = 10) // 所有接口都10次/分钟
```

### 3. 监控告警原则
```yaml
设置告警:
  - 连接池使用率 > 80%
  - 缓存命中率 < 50%
  - 接口P99响应时间 > 1s
  - 错误率 > 1%

不要:
  - 告警过于敏感（产生告警疲劳）
  - 告警不及时（问题已扩大）
```
